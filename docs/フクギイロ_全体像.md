# フクギイロ 全体像 — 24時間365日回り続ける「可処分所得を増やす機械」

作成日: 2026-07-22 / 位置づけ: 全部門・全文書の最上位イメージ(小柳さんの構想を機械の設計図に翻訳したもの)
一言で: **眠らないメディアで県民に出会い、診断で「もらい忘れ」を見つけ、申請を限界まで簡素化して入金まで見届け、「我々が関わって増えた可処分所得」をデータとして積み上げ、その信頼の上でB2Bマネタイズする。**

## 実感ファースト宣言(2026-07-23 小柳さん決裁・最上位原則)
- マネタイズは「県民がちゃんとプラスを実感し、多くの人が実感してから」。収益指標が実感指標より先に立ったら設計ミスと判定する
- 県民からは一切取らない(恒久)
- 最終目標: **県民の10% ≒ 14万人の登録**。そこまで届いたとき、「県民がプラスになる商品を持つ企業のみに広告枠を売る」誰も損しないモデルが出口に加わる(審査基準策定+CV単一原則の改定決裁が実装条件。詳細: docs/フクギイロ_計測レポート設計.md §5)
- 指標の序列: 受給実感 > 登録 > 行動 > 認知 > 収益

        ┌─────────────── 眠らない集客層(L3〜L4自動) ───────────────┐

この円環が1周するたびに、(a)県民の可処分所得 (b)データ資産 (c)収益 の3つが同時に増える。**どれか1つだけ増える施策は円環のどこかが壊れている**と判定する。

## 2. 「入金まで」の設計 — 合法の範囲で限界まで簡素化する

| やること | やらないこと(法の線) |
|---|---|
| 必要書類のチェックリスト・事前整理シートの自動生成 | 申請書類の作成・提出の代行(行政書士法・社労士法) |
| 窓口・期限・持ち物のナビゲーション | 本人に代わる役所とのやりとり |
| 「申請した?」「振り込まれた?」の自動フォロー | 受給額に連動した報酬の受領 |
| 専門判断が要る案件の士業へのワンタップ接続 | 受給可否の断定 |

- 将来、行政側のオンライン申請(ぴったりサービス等)が拡充されるほど「事前整理シート→そのまま入力」の距離が縮まる。**行政のDXが進むほどフクギイロの伴走価値は上がる**設計
- 最終目標は行政自身のプッシュ型給付化(申請そのものの消滅)。その日までの橋を架けるのがフクギイロ

## 3. 計測の設計 — 「我々が関わって増えた可処分所得」の定義

- **世帯単位**: 受給報告(自己申告)ベースで「制度名×金額×受給日」を記録。カウンターは検証済み報告のみ計上(トドケ管理・基準変更はニドナシ機構も監視)
- **総額**: サイトに常時公開(実受給額カウンター)。月次で「沖縄もらい忘れレポート」として市町村別・制度別に統計化
- **反実仮想の扱いに正直であること**: 「我々が関わらなくても受給できた世帯」も混ざる。だから対外的には「フクギイロ経由の受給確認額」と表記し、「我々が増やした額」と断定しない(正確性最優先)。関与効果の推定は診断時点の「知らなかった」回答率で補助的に示す
- 目標値: 1年目1,000世帯×平均12万円=1.2億円 → 3年目1万世帯12億円 → 最終的に県平均可処分所得+月1万円へ政策接続
