# フクギイロ 世帯診断フロー設計 v1 — ミチビキさん起案

状態: 設計(LINE実装前。ロジックはWeb版・LINE版共通で使う) / 前提: 個人情報保護方針ドラフト準拠(取得最小化・匿名可)
目標: **3分・最大7問**で「あなたの世帯が使える可能性のある制度と概算」を出す。診断完了→LINE登録率30%。

## 質問ツリー(v1)

```
Q1. お住まいの市町村は?(選択: 41市町村+「県外」)
    → 県外なら: 「沖縄県内の方向けのサービスです」+国の制度のみ表示
Q2. ご家族は?(大人の人数 / 子どもの人数)
Q3. お子さんの年齢は?(未就学 / 小学生 / 中学生 / 高校生 / 大学・専門 / 複数選択)
    → 子ども0人ならスキップ
Q4. 最近あった・これからある出来事は?(複数選択)
    妊娠・出産 / 入園・入学 / 引越・住宅購入 / 就職・転職 / 失業 / 病気・けが / 介護 / 災害 / 特になし
Q5. 世帯の収入はざっくりどのくらい?(任意・5区分)
    ※「答えなくても診断できます」を必ず表示(ウタガイ採用事項)
Q6. 住まいは?(持ち家 / 賃貸 / 実家 / 公営住宅)
Q7. (Q4で失業/病気/介護を選んだ場合のみ)雇用保険・健康保険の加入は?(会社員 / 自営・フリーランス / パート / 無職)
```

## 出力様式(断定禁止の徹底)

```
【診断結果】あなたの世帯が使える可能性のある制度: ◯件
もらい忘れ概算: 最大 ◯◯円/年 の可能性
─────
■ 児童手当(全国)
  対象の可能性: 高 / 概算: 月◯円〜(お子さんの年齢による)
  次の一歩: ◯◯市役所 こども課 → [公式ページ]
■ …
─────
※ この診断は公式情報に基づく「可能性のご案内」です。受給できるかの最終判断は各窓口で行われます。
※ 概算は目安です。正確な金額は公式ページでご確認ください。
```

- 概算の合計は「最大〜の可能性」表記固定。制度ごとに「対象の可能性: 高/中/要確認」の3段階(「確実」は使わない)
- 医療・障がい・介護系は金額を出さず「窓口案内」のみ(守り部方針)
- 診断結果の下にLINE登録導線: 「締切が近づいたらお知らせします(無料)」

## マッチングロジック
- 制度DB(data/fukugiiro/seido.json)の life_events / area / target_household を条件式にコンパイル
- **条件式はClaude Codeが制度DBから自動生成し、ケンショウ承認のevalに合格したものだけデプロイ**(L2)
- 診断ロジックのバージョンはDBの更新日と紐付け(古いDBで診断しない)

## eval(トラジェクトリテスト)設計
| ケース種別 | 例 | 期待 |
|---|---|---|
| 正例 | 那覇市・子2人(小/中)・収入帯低 | 児童手当・就学援助系が「高」で出る |
| 負例 | 子0人 | 子育て制度が出ない |
| 境界 | 高校生のみ | 児童手当が出ない(中学生まで)※制度改定に追従 |
| 地域 | 県外 | 市町村制度が出ない |
| 表現 | 全ケース | 出力に禁止表現(必ず/確実/審査なし)ゼロ |
- ケースはDB更新のたびに自動再生成し、CIで実行(fukugiiro-ci.ymlに追加予定)

## 実装順序(LINE開設に依存しない)
1. **Web版診断**(site内・静的+JS)を先に作る → サイト公開(M1 Week4)と同時に診断も動く
2. LINE版は同じロジックJSONを読む(開設後に接続するだけの状態にしておく)
3. 診断ログ(匿名・回答パターンのみ)で完了率・離脱点を計測

## 三名体制議論の記録
- スイシン: Web版先行はLINE開設待ちを消す良手。7問3分は妥当
- ウタガイ: Q5(収入)とQ7(保険)は離脱リスクが最も高い。Q5任意は良いが、Q7も「わからない」選択肢が必要 → 採用
- ベッカイ: 質問を減らす別解として「ライフイベント1つだけ選んで即3件表示→深掘りは後」のクイック診断も併設できる。完了率が低ければこちらを主に → 採用(v1.1のA/B候補として登録)
- 裁定(ミチビキ): v1は7問フル診断で開始、計測開始後1ヶ月でクイック診断とのA/B判断
