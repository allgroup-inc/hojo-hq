# フクギイロ 個人向け制度データスキーマ v1

対象ファイル: data/fukugiiro/seido.json / 検証: scripts/validate_fukugiiro.py(CIで自動実行)
原則: **verified=false の項目はサイトに「要確認」表示でのみ掲載可**。断定表現は検証スクリプトが機械的に拒否する。

## トップレベル
| キー | 型 | 説明 |
|---|---|---|
| updated_at | str | 最終更新(JST, YYYY-MM-DD HH:MM) |
| count | int | items件数(検証で自動照合) |
| items | list | 制度エントリの配列 |

## 制度エントリ
| キー | 型 | 必須 | 説明 |
|---|---|---|---|
| id | str | ○ | `fk-{発行元区分}-{slug}`(例: fk-kuni-jidoteate / fk-naha-xxx)。一意 |
| name | str | ○ | 制度の正式名称(通称は notes へ) |
| category | str | ○ | 子育て / 住まい / 医療・健康 / 教育 / 仕事・失業 / 生活支援 / 介護 / 防災・その他 |
| life_events | list[str] | ○ | 妊娠・出産 / 子育て / 入園・入学 / 就職・転職 / 失業 / 住宅取得・引越 / 病気・けが / 介護 / 障がい / 災害 / 低所得・生活苦 |
| issuer | str | ○ | 発行元(こども家庭庁・沖縄県・那覇市 等) |
| area | str | ○ | 全国 / 沖縄県 / 市町村名 |
| target_household | str | ○ | 対象世帯の平易な説明(断定表現禁止) |
| amount_note | str | ○ | 金額の説明。未検証なら「要確認」 |
| deadline_type | str | ○ | 常時 / 期限あり / 年度内 / 要確認 |
| deadline | str/null | 条件 | deadline_type=期限あり の場合必須(YYYY-MM-DD) |
| how_to_apply | str | ○ | 申請窓口・方法(市区町村窓口 / オンライン 等) |
| source_url | str | ○ | 一次ソースURL(https必須。go.jp / lg.jp / pref.okinawa.jp 等の公的ドメインを推奨、それ以外は警告) |
| verified | bool | ○ | ケンショウ(検証部)が一次ソースと突合済みか |
| verified_at | str/null | 条件 | verified=true の場合必須 |
| verified_by | str/null | 条件 | verified=true の場合必須(kensho / 人間名) |
| status | str | ○ | 検証済み / 要確認 / 終了。verified=false なら「要確認」固定 |
| notes | str | — | 補足(断定表現禁止) |

## 機械検証ルール(validate_fukugiiro.py)
1. 必須フィールド・型・カテゴリ/ライフイベントの語彙チェック
2. id一意・count一致・source_urlのhttps必須(非公的ドメインは警告)
3. deadline_type=期限あり なのに deadline が無い/過去日 → エラー(過去日は status=終了 のみ許容)
4. verified=true なのに verified_at/verified_by が無い → エラー / verified=false なのに status=検証済み → エラー
5. **禁止表現検査**: 「必ずもらえる」「絶対」「審査なし」「誰でももらえる」「100%」等が name/target_household/amount_note/notes に含まれたらエラー(マモリさんの禁止語リスト: スクリプト内 FORBIDDEN)
6. ゴールデンセット自己テスト(`--self-test`): tests/golden_fukugiiro.json の正例が通り・負例が落ちることをCIで毎回確認(検証器自体の劣化を検知)

## 運用
- 追加・変更はPR経由。CIの検証が通らないものはマージ不可(=公開されない)
- 禁止語リストの変更はマモリさん審査+うちむき担当の承認(L2)
- スキーマ変更は三名体制議論+ケンショウ承認の上でこの文書と検証器を同時に更新
