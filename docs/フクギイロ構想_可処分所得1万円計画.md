## 0. ミッションの正直な数値化(ここから逃げない)

**「沖縄県民の可処分所得を平均で月1万円上げる」の本当のサイズ:**

- 沖縄県の1人当たり県民所得は約217万円で全国最下位(全国平均の約7割)。世帯実収入は全国平均を年約41.5万円下回る
- 県民約145万人の平均可処分所得を月1万円動かす = **年間約1,700億円** の所得移転に相当
- これは夫婦2人+PCが「直接配れる」金額ではない。**しかし「受け取れるはずのお金が申請主義のせいで届いていない」構造(未申請問題)を突けば、行政・金融機関・士業を巻き込んだ「受給率を上げる社会インフラ」になることで到達可能な目標**である

よってKGIを2層に分ける:

| 層 | 指標 | 定義 |
|---|---|---|
| **KGI-1(事業KGI)** | フクギイロ経由の「実受給・実軽減額」累計 | 利用者が実際に口座で受け取った給付・補助・減免額。自己申告+受給報告で計測 |
| **KGI-2(社会KGI)** | 支援世帯あたり **年12万円(=月1万円相当)** の実受給 × 世帯数 | 1年目1,000世帯(1.2億円) → 3年目1万世帯(12億円) → 10年目「行政のプッシュ型給付化」への政策接続で県全体へ |

**核心の差別化 = 「情報を載せる」で終わらず「振り込まれた」まで追う。** 世界中の類似サービスのほぼ全てが「情報提供」で止まっている。受給完了率をKPIにしたプレイヤーは国内に存在しない。ここが「誰も成し遂げていない」部分の正体。

## 2. 3つの専門家パネルからの統合見解

> 注記(正確性最優先ルール): 以下は「実在の100人にインタビューした結果」ではない。上記の実在事例・公開情報のリサーチに基づき、3つの視点(情報収集のプロ/Claude Code活用の少人数経営者/手残り月200万到達経営者)を統合したシミュレーション壁打ちである。実名の声が必要な局面(金融機関提携等)では士業ネットワーク・商工会経由で本物のヒアリングを行うこと。

**パネルA: 情報収集のプロ100人の視点(要約)**
1. 勝負は「網羅」ではなく**鮮度と検証**。締切切れ情報を1件載せた瞬間に信頼が死ぬ(hojo-hq検証部の思想は正しい)
2. 個人向けは企業向けと違い**ライフイベント起点**で引く(出産・入学・介護・失業・住宅・病気)。制度名で探す県民はいない
3. 41市町村の広報紙PDF・LINE・防災無線アプリまでが情報源。**API化されていない情報こそ価値**
4. 一次ソースURLを常に添付。転載サイトを信じない
5. 収集は自動化、**判断(これは載せるか)は人間**に残す

**パネルB: Claude Codeで夫婦経営する100人の視点(要約)**
1. 人を雇う代わりに**エージェント部門を雇う**。hojo-hqの8部門体制(アカリ/シュシュ/ケンショウ/マモリ/ヒロメ/ツナグ/ムスブ/タダス)はまさにこの形。フクギイロ側に個人給付版の部門を増設すればよい
2. 夫婦の役割は「承認とアポ」だけに絞る。**人間がやるのは、判断・面談・謝罪・決裁の4つ**
3. GitHub Actions + Claude APIの無人パイプライン(hojo-hqで稼働済み)が最大の資産。限界費用ほぼゼロで個人制度に横展開できる
4. 週次レポ・KPI集計・投稿生成は全自動化し、夫婦の可処分「時間」をまず増やす
5. 落とし穴は「自動化の自動化」に溺れること。**月次で「受給できた県民の実名の声」を必ず1件取りに行く**。これだけは自動化しない

**パネルC: 1年後に夫婦手残り月200万の経営者100人の視点(要約)**
1. 手残り月200万 = 法人粗利で**月280〜350万円**が必要(役員報酬2人分+社保+法人維持費)。逆算で設計する
2. 到達者の共通点は「**単価の高いB2B収益 × 集客装置としての無料B2C**」。B2Cだけで200万に届いた者はいない
3. 最初の12ヶ月は「1本柱に集中→2本目は黒字化後」。柱を最初から5本立てた者は全員死んだ
4. **既存の信頼資産(ALLGROUPの保険顧客・催事・GLOW・士業網)から最初の100件を取る**。ゼロから広告で始めない
5. 価格は「もらえた金額の◯%」ではなく(士業法リスク)、**定額の法人課金と送客手数料**に置く

## 3. 事業モデル: フクギイロの構造

### ブランド
- 社名: **株式会社フクギイロ**(フクギ=沖縄の集落を台風から守る防風林「福木」。県民の家計を守る木、の意。福木の花言葉的な「豊かさの黄色」をブランド色候補に)
- 関係: 「沖縄企業のミカタ」(企業向け・hojo-hq)の姉妹ブランド。**企業のミカタ=B2B、フクギイロ=B2C(県民世帯向け)**
- CV: LINE登録1点(企業のミカタと同思想。ただし別アカウント)

### ファネル(「実際に手にする」までの5段)

```
①知る    Instagram/検索/紹介 →「あなたの世帯、いくらもらい忘れてる?」
②診断    LINE上で世帯診断(家族構成・年収帯・ライフイベント)→ 対象制度と概算額を提示
③準備    必要書類チェックリスト+申請窓口+締切リマインドを自動配信
④申請    本人申請の伴走(代行はしない)。専門判断が要る案件は士業へ接続
⑤受給    「振り込まれた」報告をLINEで回収 → 実受給額カウンターに積算・公開
```

⑤が全ての肝。**トップページに「フクギイロ経由で県民が受け取った金額: ◯◯円」のリアルタイムカウンター**を置く。これがPRであり、KGIであり、行政・金融機関への提案書になる。

### 収益モデル(夫婦手残り月200万への逆算)

利用する県民からは**一円も取らない**(Propel/Civichatの教訓)。収益は法人側から。

| 柱 | 内容 | 想定単価 | 1年後の目標 |
|---|---|---|---|
| 1. 企業向け送客(既存) | 企業のミカタ経由の士業送客手数料 | 1件3〜10万円 | 月80万円 |
| 2. 法人サブスク | 企業向け「使える制度アラート+個社マッチングレポート」 | 月1〜3万円×法人 | 60社=月120万円 |
| 3. 金融機関・商工会OEM | 沖縄の銀行・信金・商工会に診断ツール/DBを提供(実受給実績カウンターを提案書の武器に、公開情報+実績ベースで直接開拓) | 月10〜30万円×機関 | 2機関=月40万円 |
| 4. ALLGROUPクロス | 診断で家計課題が見えた世帯→「家計の見直しやさん」FP相談へ(本人同意制) | グループ内評価 | 月40万円相当 |
| 5. 講演・研修・取材 | 「AI×夫婦2人で県民運動」の事例価値そのもの | 1本5〜15万円 | 月20万円 |

合計 月300万円粗利 → 手残り夫婦200万円。**1年目は柱1・2に集中**(パネルC-3)。柱3は6ヶ月目から、柱4はCV単一原則との整合を小柳さん決裁で設計してから。

### 法務の線引き(マモリ絶対遵守)
- **申請代行はしない**: 雇用系助成金=社労士独占、官公署提出書類作成=行政書士独占。フクギイロは「情報・診断・リマインド・窓口接続」まで
- 成果報酬(受給額の◯%)は取らない。定額課金と送客のみ
- 世帯診断は個人情報の塊。取得項目最小化・同意設計・保管方針を守り部が事前審査
- 「もらえます」と断定しない。概算は「〜の可能性」表記、最終判断は必ず一次ソースと窓口へ

## 4. 成功までの5ステップ

### STEP 1(0〜2ヶ月): 基盤移植 — 「企業のミカタ」の技術資産を個人向けに複製
- 株式会社フクギイロ設立(定款の事業目的に情報提供業・広告業・業務受託を明記。士業業務は入れない)
- hojo-hqのパイプライン(Actions cron→Claude API構造化→静的公開)をフォークし、**個人向け制度DB**を新設。データスキーマに「ライフイベント」「世帯要件」「概算額」軸を追加
- 対象制度の初期棚卸し: 国(児童手当・住宅・高等教育無償化等)+県+まず**那覇・沖縄・うるま・浦添・宜野湾の5市**から
- エージェント組織増設: 個人給付版の収集・検証・守りを既存部門の管轄拡張として定義
- **完了条件: 個人向け制度100件が検証済みで公開されている**

### STEP 2(2〜4ヶ月): 診断ファネル構築 — 「いくらもらい忘れてるか」が3分でわかる
- LINE公式アカウント(フクギイロ)+世帯診断フロー(Turn2us型)。診断ロジックはClaude Codeで制度DBから自動生成し、ケンショウ(検証部)が突合
- 締切リマインド・書類チェックリスト自動配信(ツナグの設計を転用)
- **受給報告の仕組み**(⑤)をローンチ時から実装。カウンターをサイトに掲示
- 最初の100世帯はALLGROUP既存顧客(保険・催事)から。広告は使わない
- **完了条件: 診断完了→LINE登録率30%以上、最初の「受給できました」報告10件**

### STEP 3(4〜7ヶ月): マネタイズ点火 — 法人側から収益を立てる
- 企業のミカタ側の士業送客をオペレーション化(単価・件数のトラッキング)
- 法人サブスク(制度アラート+個社マッチングレポート、レポートはClaude自動生成+人間承認)を20社に導入。初期顧客はムスブ経由の既存基盤から
- 受給実績カウンターを武器に地元メディア・県広報へプレスリリース
- **完了条件: 月商100万円、支援世帯500、実受給額累計3,000万円**

### STEP 4(7〜12ヶ月): 面の拡大 — 41市町村と金融機関
- クローラーを41市町村全域へ(広報紙PDF含む。robots.txt/規約審査は守り部)
- 金融機関・商工会OEM提案。入口は①商工会・士業ネットワークからの紹介、②受給実績カウンター+匿名統計を添えた直接提案の2ルート。「窓口で世帯診断を使ってもらう」形で受給率向上を共同事業化
- Instagram×ヒロメ: 「今月のもらい忘れワースト5」等の定型シリーズ自動生成
- **完了条件: 月商300万円(=手残り200万円ライン)、支援世帯1,000、実受給額累計1.2億円**

### STEP 5(12ヶ月〜): 県民運動化 — 「申請主義」自体を変えに行く
- 実受給データ(どの制度が、どの属性に、なぜ届いていなかったか)を匿名統計化し、県・市町村に**プッシュ型給付への政策提言**として提供
- 目標を「フクギイロが配る」から「**行政が取りこぼさなくなる**」へ引き上げる。国もプッシュ型給付(特定公的給付)の制度整備を進めており、追い風は実在する
- 3年で1万世帯×年12万円=年12億円、10年で「受給率で沖縄が全国トップ」を県の公式目標に押し上げることが最終ゴール
- **ここまで来て初めて「県民平均の可処分所得を動かした」と言える**

## 6. リスクと要確認事項

1. **要確認(最優先)**: 収益柱4(ALLGROUPクロス)は「CV単一=LINE登録のみ」の絶対ルール4に関わる。フクギイロを別ブランド・別LINEとして切り出す前提で、導線設計を小柳さん決裁にかける
2. **要確認**: 世帯診断の個人情報取扱い(プライバシーポリシー・保管・グループ内共有の同意設計)は守り部審査+必要なら弁護士確認
3. 受給報告(⑤)の回収率が低いとKGIが測れない → 報告インセンティブ(お礼スタンプ・抽選等の景表法確認込み)を設計
4. 市町村サイトのクロール可否は1自治体ずつ守り部承認(既存ルール踏襲)。不可の自治体は「リンク+手動更新」で運用
5. 「妻2人とPC」は「夫婦2人+PC」体制として本書を作成。人員前提が異なる場合は再設計
