## 4. 進め方(三名体制議論済みの前提で運用)
1. マモリさんが#1〜#19のrobots.txt/利用規約を審査 → 結果を docs/守り部審査記録.md に追記
2. 承認されたソースから、シュシュが個人向けスキーマ(docs/フクギイロ_データスキーマ.md)で構造化
3. すべて verified=false・status=要確認 で入り、ケンショウが一次ソース突合後に verified=true 化
4. CI(fukugiiro-ci.yml)が通らないデータはマージ不可
5. ドメイン・URLはクロール前にマモリさんが実在・正当性を確認する(本リストの表記は入口の見当であり、確定URLは審査記録を正とする)

## 5. ウタガイ役の記録(三名体制)
- 懸念: 個人向け制度は「随時の臨時給付」が多く、国の入口ページだけでは取りこぼす。→ 対策: 首相官邸・デジタル庁の給付関連特設と、県の広報(既に承認済みの範囲)を横断チェックする運用をM2で追加検討
- 懸念: 医療・障害・介護は誤案内の影響が重大。→ 対策: この3領域は当面「診断で出さない」(サイト掲載のみ、status=要確認固定)から始め、専門家レビュー後に診断対象へ昇格
