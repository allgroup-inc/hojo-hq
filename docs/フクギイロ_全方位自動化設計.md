# フクギイロ 全方位自動化設計 — 経営12機能 × 世界の実証パターン × Claude Code実装

作成日: 2026-07-22 / 親文書: 構想書・月次実行計画・自動化経営の方程式
位置づけ: 「達成の方程式」(7本)を経営の全機能に展開した実装仕様。各機能に「世界の実証 → フクギイロ実装 → 自律度 → 導入月」を定義する。

自律度表記は方程式3のダイヤルに準拠: L4=全自動 / L3=自動+事後監査 / L2=人間承認後実行 / L1=人間のみ

---

## 機能1: 集客(獲得)
- **世界の実証**: 無料ミニツール+programmatic SEO+build in publicの三点セットが少人数事業の獲得定石(Marc Lou/Levels)。広告に頼らずCACほぼゼロを実現
- **実装**:
  - 世帯診断(無料)を最上流に。診断結果ページから「もらい忘れ額」をシェアできる画像をCanva API/デザインパイプラインで自動生成(拡散装置)
  - programmatic SEO: 制度×市町村×ライフイベントのページをClaude Codeが生成・毎日更新。タイトル・構造化データ(FAQ/HowToスキーマ)も自動
  - Instagram: ヒロメの投稿パイプラインを個人向けに拡張(「今月のもらい忘れワースト5」等の定型シリーズ)。UTMで診断流入を計測
- **自律度**: ページ生成L3 / SNS投稿L2 / 診断ロジックL2
- **導入**: M3〜M4(SEOはM4〜)

## 機能2: 販売(法人営業)
- **世界の実証**: 少人数経営の営業は「人間はアポと面談だけ、準備と追客はエージェント」型が確立
- **実装**:
  - 商談前ブリーフ自動生成: 訪問先法人の業種・規模から「その会社が使える制度リスト」をClaude Codeが生成 → 商談の手土産が毎回ある状態を作る(これ自体がサブスクのデモ)
  - 提案書・見積・商談メモのテンプレ生成、商談後のフォローメール下書き自動化
  - パイプライン管理: 商談状況をリポジトリ管理し、放置案件を週次レポで自動検知
- **自律度**: ブリーフ/下書きL4(送信はL2) / 契約L1
- **導入**: M5〜(サブスク営業開始と同時)

## 機能3: 価格(プライシング)
- **世界の実証**: マイクロSaaSの実勢価格は月$9〜49が主戦場。段階制(無料→低額→上位)+年払い割引が標準。価格は「上げてから謝る」方が「安すぎて死ぬ」より回復可能
- **実装**:
  - 県民=無料(絶対)。法人サブスクは3段: ライト月1万(アラートのみ)/スタンダード月2万(個社レポート月1回)/プロ月3万(レポート+診断ウィジェット設置権)
  - βのM5は50%価格で10社→M7に正価へ。解約理由は全件記録し、四半期ごとに価格改定を小柳さん決裁で判断
- **自律度**: 価格分析L4 / 改定L1
- **導入**: M5(β)→M7(正価)

## 機能4: 顧客対応(CS)
- **世界の実証**: Claude搭載のCS AIは定型問合せの6割前後を自動解決(Tidio Lyro実測67%)。2026年のCS AI導入率は66%に到達。LINEを含むマルチチャネル統合が標準
- **実装**:
  - LINE上のFAQ自動応答(制度DB+FAQをソースに)。**ただし「あなたはもらえるか」の個別断定はさせない**(守り部ルール: 診断フローか窓口へ誘導する回答に制限)
  - 回答不能・怒り・個人事情の相談は即・人間エスカレーション(エスカレーション条件をhooksで強制)
  - 全応答ログを週次でタダス(監査)がサンプリング監査
- **自律度**: FAQ L3 / 個別相談L1 / エスカレーション設計L2
- **導入**: M4〜

## 機能5: 経理・バックオフィス
- **世界の実証**: 2026年、freee・マネーフォワードとも**公式MCPサーバーを全プラン開放**(freee-mcpは2026年3月〜)。Claudeから自然言語で仕訳登録・照会・帳票出力が可能に。「SaaS+AI」の3層構成で請求〜月次決算の工数を半分以下にした事例が国内に多数
- **実装**:
  - freee(またはMF)+公式MCP接続で、月次: 請求書発行→入金消込→仕訳→月次レポートまでをClaude Code定期タスク化。人間は月次決算の承認のみ
  - サブスク請求は自動課金(Stripe等)にし、未入金検知→督促下書きまで自動
  - 経費レシートはOCR自動仕訳、役員報酬・社保・納税スケジュールはリマインダー管理
- **自律度**: 記帳・照会L3 / 請求書送付L2 / 支払実行・税務判断L1(税理士+人間)
- **導入**: M1(設立と同時に仕組みから作る)

## 機能6: 法務・コンプライアンス(守り)
- **世界の実証**: エージェント運用の標準は「防御の多層化」(入力ガード→行動ゲート→出力ガード→人間承認→evalで還流)。規制業種ほど「コードで強制されたゲート」が有効
- **実装**:
  - hooksで強制: ①公開ディレクトリ書込=検証済みフラグ必須 ②配信系実行=承認ファイル必須 ③個人情報を含むパスの取扱制限
  - 士業法チェックリスト(代行・報酬形態・表現)を出力ガードのevalに組込み、「申請を代行します」等の禁止表現を自動検知
  - 個人情報: 取得項目最小化・診断データの保持期限・グループ共有の同意設計を守り部審査記録(既存様式)で管理
- **自律度**: 検知L4 / 判定L2 / 方針変更L1
- **導入**: M1〜M2(公開開始前に完成)

## 機能7: 開発・インフラ
- **世界の実証**: ヘッドレス`claude -p`+GitHub Actions cron+サブエージェント並列+リモート制御が2026年の無人運用標準形。夜間の依存脆弱性スキャン→自動PRのような「無人保守」も月額数十ドル水準で実現
- **実装**:
  - hojo-hq基盤の移植(実証済み)。市町村別サブエージェント並列収集(split-and-merge)
  - 夜間ジョブ: リンク切れ検査・締切超過制度の自動アーカイブ・依存更新PR・バックアップ
  - すべての変更はPR経由: 生成エージェントがPR作成→検証エージェントがレビュー→人間が金曜マージ(コードもデータも同じ統制)
- **自律度**: 保守L3 / 本番反映L2
- **導入**: M1〜継続

## 機能8: データ資産(モート)
- **世界の実証**: 2026年の定説は「データは貯めるだけではモートにならない。**製品を良くするループに還流して初めてモート**」。防御力の源泉はワークフロー深度・独自データループ・流通・スイッチングコスト
- **実装**: フクギイロのデータループを明文化:
  1. 診断利用→「どの属性が何を知らないか」が蓄積→診断の的中率が上がる
  2. 受給報告→「実際に通った申請の傾向」が蓄積→案内の精度と説得力が上がる
  3. 法人レポート利用→「企業が本当に使う制度」が見える→営業ブリーフが強くなる
  - このループ(受給実績データ)は**沖縄では他の誰も持っておらず、後発が買えない**。匿名統計の行政提供・金融機関ライセンスは「データ事業化」の実証済みパターン
  - 注意: DBの制度情報そのものは公開情報でありモートではない。**モートは「受給完了までの伴走ワークフロー+実績ループ」**。だからこそ指針1(受給完了が唯一の真実)が経済的にも正しい
- **自律度**: 集計L4 / 外部提供L1
- **導入**: M3(受給報告開始)〜

## 機能9: 広報・ブランド
- **世界の実証**: build in publicの本質は「数字+物語+頻度」。数字だけでは拡散しない(アンチパターン6)
- **実装**:
  - 月次「沖縄もらい忘れレポート」を自動生成し、県内メディアへ定期配信(データジャーナリズム素材の提供者になる)
  - 受給者の実名の声(月1・人間が取材)を核に、SNS・プレス・提案書へClaude Codeが多面展開
  - 危機管理: 誤情報事故時の公表文テンプレと手順を平時に整備(L1運用)
- **自律度**: 素材生成L2 / 取材・謝罪L1
- **導入**: M7〜(実績が貯まってから)

## 機能10: 提携(パートナーシップ)
- **世界の実証**: 少人数事業の提携は「相手の業務に埋め込まれる」ほど強い(ワークフロー深度=スイッチングコスト)
- **実装**:
  - 士業向け: 送客だけでなく「紹介案件の事前整理シート」(世帯/企業の状況+該当制度+必要書類)を自動生成して渡す → 士業の工数を削る存在になる=切られない
  - 金融機関/商工会OEM: 診断ウィジェット+月次利用統計レポートを自動納品。導入先の窓口研修資料もClaude Codeで生成
  - 提携先ごとの成果(送客数・成約・受給額)ダッシュボードを自動更新し、四半期レビューを定例化
- **自律度**: 資料・統計L3 / 提携条件L1
- **導入**: M6(士業定型化)→M9〜(OEM)

## 機能11: 監査・ガバナンス
- **世界の実証**: エージェント運用の成熟組織は「観測性→評価→方針更新」のライフサイクル(ADLC)を常設し、トラジェクトリ(過程)まで評価する
- **実装**:
  - タダス(独立監査・アカリ指揮系統外)が毎週: 公開データサンプル照合・配信ログ・CS応答・経理仕訳をサンプリング監査し、指摘をeval/テストケース化して還流
  - 月次で「自律度昇格/降格審査」(L2→L3昇格はeval合格率+事故ゼロ3ヶ月)
  - すべてのエージェント判断はログ保存(リポジトリ=監査証跡)。小柳さん・たかしくんはダッシュボードでいつでも閲覧可能
- **自律度**: 監査実行L3 / 処分・方針L1
- **導入**: M1〜継続

## 機能12: 組織・企業価値
- **世界の実証**: ソロ/少人数AI企業の企業価値は収益の10〜50倍で取引される例があり(2026年、ステージ次第)、Base44型の「小さく作って高く売る」出口も現実化。ただし評価されるのは売上でなく「再現可能なワークフローとデータループ」
- **実装**:
  - フクギイロのエージェント組織図(hojo-hq 8部門を継承・個人給付管轄を拡張)と全ワークフローをリポジトリに文書化し続ける = **事業自体を「引き継ぎ可能な資産」として構築**
  - 出口は売却が目的ではないが、「行政・金融機関との共同事業化」「グループ内での事業価値評価」の際に、文書化された自動化経営そのものが交渉力になる
  - 夫婦の後継・不測時対応: 全オペレーションが文書+コードであるため、事業継続計画(BCP)が最初から内蔵されている
- **自律度**: 文書化L3 / 組織変更L1
- **導入**: M1〜継続(文書化は日常業務の副産物として自動生成)
