# 株式会社フクギイロ 行動指針・月次実行計画(12ヶ月)

作成日: 2026-07-22 / 親文書: docs/フクギイロ構想_可処分所得1万円計画.md
前提: 夫婦2人+PC+Claude Code。嶺井さん等の個人人脈には依存しない設計。最終決裁は小柳さん。

## 第1部 行動指針(10ヶ条)

1. **受給完了が唯一の真実。** 登録数・PV・フォロワーは中間指標。「振り込まれた」報告と実受給額カウンターだけを成果と呼ぶ。カウンターが伸びない週は、施策のどこかが間違っている
2. **正確性はスピードに勝つ。** 締切・金額・要件は一次ソース照合済みのみ公開。不明は「要確認」表示。誤情報1件は信頼全損
3. **県民からは一円も取らない。** 収益は常に法人側(士業・法人サブスク・金融機関)から。Propelの成功とCivichatの失敗が示した鉄則
4. **線を越えない。** 申請代行はしない(社労士法・行政書士法)。診断・書類チェックリスト・リマインド・窓口/士業接続まで。成果報酬(受給額の%)も取らない
5. **人間の仕事は4つだけ。** 判断・面談・謝罪・決裁。それ以外(収集・検証・生成・配信・集計・レポート)はすべてClaude Codeとエージェント部門に任せ、任せられない作業は「やらない」ことを検討する
6. **1年目の収益は2本柱に集中。** 柱1(士業送客)と柱2(法人サブスク)のみ。柱3(金融機関OEM)は実績カウンターが武器になるM9まで営業しない。柱を増やすのは黒字化後
7. **最初の100件は信頼資産から。** ALLGROUP既存顧客(保険・催事・GLOW)と紹介のみで立ち上げる。広告費はゼロ。広告解禁は小柳さん決裁事項
8. **月1件、実名の声を取りに行く。** 「受給できた県民の顔と言葉」だけは自動化しない。これが提案書・SNS・政策提言すべての燃料になる
9. **数字は週次で見て、方針は月次でしか変えない。** 日次の上下で右往左往しない。月末レビューで「クリア条件」に照らして続行/修正/撤退を判断する
10. **エージェントは提案まで、決裁は小柳さん。** 予算・提携・掲載方針・導線追加はすべて上申(hojo-hq絶対ルール5を継承)

## 第3部 月次実行計画

数値は累計。月商=フクギイロ+企業のミカタ経由の粗利ベース。

### M1(1ヶ月目)— 設立と土台移植
**テーマ: 会社をつくり、hojo-hqを個人向けに複製する**
- 人間: 株式会社フクギイロ設立手続き(定款目的: 情報提供業・広告業・業務受託。士業業務は入れない)。法人口座・会計freee等の整備。守り部の個人情報方針ドラフトを審査・決裁上申
- Claude Code: hojo-hqパイプラインをフォークし個人向け制度DB新設。スキーマに「ライフイベント・世帯要件・概算額」軸を追加。国の個人向け制度(児童手当・住宅・教育・医療系)の構造化開始
- **クリア条件: 法人設立完了 / 個人向け制度50件を検証済みで公開 / 個人情報方針の決裁完了**
- 数値: 月商0円

### M2(2ヶ月目)— 制度100件と入口開設
**テーマ: 「沖縄で一番正確な個人向け給付DB」の初版**
- 人間: 那覇・沖縄・うるま・浦添・宜野湾の5市分のクロール可否を守り部審査で確定(不可自治体はリンク+手動運用)。フクギイロLINE公式アカウント開設。ブランド(ロゴ・配色)を小柳さん決裁
- Claude Code: 5市クローラー生成。二重検証系(生成と検証を別エージェント)稼働。サイト初版実装
- **クリア条件: 検証済み100件公開 / 5市カバー / LINEアカウント稼働**
- 数値: 月商0円

### M3(3ヶ月目)— 診断ローンチと最初の100世帯
**テーマ: 「あなたの世帯、いくらもらい忘れてる?」が動き出す**
- 人間: ALLGROUP既存顧客(保険・催事)へ案内開始(提携部ムスブの導線を転用)。最初の受給者インタビュー実施。士業送客の初件を成約
- Claude Code: LINE世帯診断フロー実装(制度DBから診断ロジック+テストケース自動生成、ケンショウ突合)。受給報告フォームと実受給額カウンターをサイトに実装
- **クリア条件: 診断ローンチ / LINE登録150世帯 / 受給報告3件・実受給額100万円 / 送客初成約**
- 数値: 月商10万円

### M4(4ヶ月目)— ファネルの数値検証
**テーマ: 診断→登録30%の壁を超える**
- 人間: 診断離脱ポイントのレビューと改善判断。紹介のお願いを既存登録世帯へ(紹介インセンティブは景表法確認込みで守り部審査)
- Claude Code: 診断UX改善のABテスト実装。締切リマインド・書類チェックリスト自動配信の本稼働。Instagram「もらい忘れシリーズ」投稿の量産開始(ヒロメ+デザイン自動化パイプライン転用)
- **クリア条件: 診断完了→LINE登録率30%以上 / 登録300世帯 / 受給報告10件・実受給額500万円**
- 数値: 月商30万円

### M5(5ヶ月目)— 法人サブスクβ
**テーマ: 収益柱2の点火**
- 人間: 法人サブスク(制度アラート+個社マッチングレポート、月1〜3万円)をβ価格で既存企業顧客10社に提案・成約。価格と解約理由を毎件記録
- Claude Code: 個社マッチングレポートの自動生成(人間承認制)。原価をほぼゼロにする
- **クリア条件: サブスク10社 / 登録450世帯 / 実受給額1,000万円**
- 数値: 月商50万円(送客20万+サブスク20万+その他)

### M6(6ヶ月目)— 半期レビューと絞り込み
**テーマ: 効いている施策1つに寄せる**
- 人間: 半期レビュー(何が世帯登録と受給報告を生んだかを特定し、下位施策を止める)。士業ネットワークとの送客オペ定型化(単価・件数・品質のトラッキング)
- Claude Code: 市町村カバーを10自治体へ拡大。受給データの匿名統計(どの制度が届いていないか)初版
- **クリア条件: サブスク20社 / 送客月5件 / 登録600世帯・支援世帯300 / 実受給額2,000万円**
- 数値: 月商80万円

### M7(7ヶ月目)— 月商100万と対外発信
**テーマ: 実績を「証拠」に変える**
- 人間: プレスリリース(実受給額カウンター+受給者の実名の声)を地元メディア・県広報へ。取材対応
- Claude Code: プレスキット・実績ダッシュボード生成。SNSで実績数値の定期発信
- **クリア条件: 月商100万円 / 支援世帯500 / 実受給額3,000万円 / メディア掲載1件**
- 数値: 月商100万円

### M8(8ヶ月目)— 面の拡大(前半)

- Claude Code: 市町村カバー20自治体。サブスクのレポート品質改善(顧客業種別テンプレート)
- **クリア条件: サブスク30社(月次解約率5%未満) / 登録750世帯 / 実受給額4,500万円**

### M9(9ヶ月目)— 41市町村完全カバーと金融機関初アプローチ

## 第5部 撤退・修正基準(先に決めておく)

- **2ヶ月連続で月次クリア条件の過半未達** → 翌月は新施策を凍結し、原因特定と修正のみに充てる
- **診断→登録率が3ヶ月連続30%未満** → ファネル設計を根本から作り直す(小手先改善を禁止)
- **M7時点で月商50万未満** → 柱2(サブスク)の価格・商品を再設計し、M9の金融機関アプローチを後ろ倒し
- **誤情報事故(締切・金額の重大誤り)発生** → 全公開停止レベルの対応を守り部主導で実施、原因公開
- 撤退・大幅転換の最終判断はすべて小柳さん決裁
