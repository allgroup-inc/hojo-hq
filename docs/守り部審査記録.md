# 守り部審査記録 — 自治体ソース収集(fetch_local.py)

- 審査日: 2026-07-22
- 審査部門: 守り部(マモリ) ／ 立会: 監査(タダス)
- 対象: `scripts/fetch_local.py` による沖縄県・関係機関サイトからの補助金/助成金情報の自動収集
- 判定: **条件付き承認(下記「遵守条件」を満たす限りにおいて可)**

## 1. 収集内容(何を取得・保持するか)
各制度について、次の**事実情報とリンクのみ**を保持する。本文(説明文)の転載は行わない。
- 制度名(タイトル)
- 締切(公募期間の終了日。判定できない場合は「要確認」)
- **原文URL(出典リンク)** — 利用者は必ず原文へ遷移して確認する
- 実施主体名(ソース名)・地域(沖縄県)

→ 掲載は「事実の索引＋原文へのリンク集」であり、著作物である説明文・写真・レイアウトの複製・転載はしない。

## 2. 対象ソースと確認結果

### (A) 沖縄県公式サイト(産業・公募補助金)
- 一覧: `https://www.pref.okinawa.lg.jp/shigoto/keizai/1009879/1010143/index.html`
- robots.txt: **存在しない(HTTP 404)** → 標準(RFC 9309)上、クロール制限の指定なし = 取得可
- 利用規約(免責事項・著作権・リンク): https://www.pref.okinawa.lg.jp/about/1018114.html
  - 著作権: 掲載情報(文章・写真・イラスト等)は著作権の対象。「引用」等**法律で認められた場合を除き無断転用・引用不可**、改変不可。
    → 本件は制度名(事実)・締切(事実)・リンクのみ保持で、説明文の転載はしないため抵触しない設計。
  - リンク: **原則フリー**。事前連絡は不要。トップページ以外へのリンクは各担当課へ相談を推奨。実施後にURL通知が望ましい(連絡先: kouhou@pref.okinawa.lg.jp)。
  - 免責: 掲載情報は予告なく変更・移転・削除されうる。→「要確認」表示と日次更新で鮮度を担保。

### (B) 公益財団法人 沖縄県産業振興公社
- 一覧: `https://okinawa-ric.jp/service/subsidy.html`
- robots.txt: `Disallow: /.shared/ /.service/ /.preview/ /.not-found/` のみ。
  → 収集対象の `/service/subsidy.html` および `/service/*.html`・`/news/entry/*` は**許可範囲**(先頭ドットの `/.service/` とは別)。
- 利用規約: 専用の利用規約/著作権ページは確認できず(プライバシーポリシー https://okinawa-ric.jp/info/privacy.html のみ)。
  → 一般の著作権法の範囲で判断。制度名・締切・リンクのみ保持し本文転載しない設計のため、リスクは低い。

## 3. 収集方法の適法・礼儀面(技術的配慮)
- robots.txt を `urllib.robotparser` で毎回確認し、Disallow のURLは取得しない。
- User-Agent を明示(連絡先付き): `hojo-hq-bot/1.0 (+https://allgroup-inc.github.io/hojo-hq; contact: bot@en-life.co.jp)`
- リクエスト間隔 **1.5秒**のレート制限。1ソース詳細取得は上限60件。
- 本文は `<main>` 要素に限定して解析し、サイドバー等の無関係情報は取得しない。
- 締切・金額が確実に取れない場合は断定せず「要確認」表示(絶対ルール#1 正確性最優先)。

## 4. 遵守条件(この条件を満たす限り承認)
1. 保持するのは**制度名・締切・実施主体・原文URL**のみ。説明文・画像の転載はしない。
2. 各制度は**必ず原文へリンク(出典明記)**する。
3. robots.txt 遵守・レート制限(≥1.5秒)・連絡先付きUAを維持する。
4. 掲載情報は「要確認」表示＋日次更新で鮮度を保つ。
5. **公開後、沖縄県広報(kouhou@pref.okinawa.lg.jp)へリンク実施を通知**する。
   ※メール送信は小柳さん/たかしくんの決裁事項。エージェントは文面提案まで。
6. 先方から掲載・リンク中止の要請があれば**即時対応(該当ソースの停止)**する。
7. 収集対象の追加・変更は、都度この記録に追記し守り部が再確認する。

## 5. 申し送り(未了・要フォロー)
- [ ] 沖縄県広報への通知メール送信(小柳さん決裁) — 文面は提携部で用意

- [ ] 那覇市・各市町村サイトを追加する場合は、当該サイトの robots.txt・利用規約を個別に本記録へ追記してから実装

# 守り部審査記録(第2号)— フクギイロ個人向けソース 第1次審査

- 審査日: 2026-07-22
- 審査部門: 守り部(マモリ)/ 立会: 監査(タダス)
- 対象: docs/フクギイロ_収集対象リスト.md の14ドメイン(国9+5市)
- 方法: robots.txt自動監査(scripts/audit_sources_fukugiiro.py、GitHub Actions実行、週次で継続監視)

## 第1次結果(robots.txt)
**全14ソース ○**(詳細: docs/フクギイロ_robots監査結果.md)
- robots.txt有り(4): こども家庭庁・厚労省・JASSO・那覇市 — いずれも収集想定パスは許可範囲
- robots.txt無し(10): RFC 9309上クロール制限の指定なし

## 判定: 🟨条件付き承認候補(クロール開始はまだ不可)
承認確定には以下が残る:
1. **利用規約・著作権ページの個別確認**(各サイトの転載/リンクポリシー)— クローラー実装前に1ソースずつ実施し本記録へ追記
2. 遵守条件は第1号(企業向け)と同一設計を適用: 事実情報(制度名・締切・窓口)+原文リンクのみ保持、説明文転載なし、連絡先付きUA、リクエスト間隔1.5秒以上、robots.txt毎回確認、中止要請即時対応
3. 医療・障害・介護の3領域は収集しても当面診断に出さない(収集対象リスト・ウタガイ懸念に基づく)

## 三名体制記録
- スイシン: 全ソース公的機関でありrobots良好。第1号と同条件で速やかに実装可
- ウタガイ: robots.txt無し=歓迎ではない。自治体サーバーは脆弱な場合があり、レート制限は既定の1.5秒より緩めず、収集は深夜帯を避ける朝夕2回に留めるべき → 採用
- ベッカイ: 国の制度はクロールよりも各府省の新着RSS・公表資料の方が確実な場合がある。実装時にRSS有無を先に確認 → 採用(実装仕様に組込み)
- 裁定(マモリ): 条件付き承認候補とし、利用規約確認の完了したソースから順次「承認」へ昇格させる
