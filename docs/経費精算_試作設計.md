# 経費精算 自動化システム 試作設計(案2: 自社構築)

作成日: 2026-07-22 / 起案: 小柳さん / ステータス: **試作設計(決裁前)**
前提: 「領収書を写真→専用宛先に送る→自動で一覧に反映→本人は目的・同席者だけ補記」の流れを、既存の道具立て(Power Automate + Claude API + Microsoft 365)で実現する。

---

## 1. 全体構成図

```
【社員のスマホ】
  領収書を撮影
      │ メール添付で送信(Phase 1)/ LINE WORKS botに送信(Phase 2)
      ▼
【共有メールボックス keihi@社内ドメイン】
      │ 新着メール(添付あり)をトリガー
      ▼
【Power Automate クラウドフロー】
  1) 添付画像を SharePoint ドキュメントライブラリへ保存(原本)
  2) 画像を Claude API(画像読取)へ送信
       → 日付/支払先/金額/税率内訳/インボイス登録番号 をJSONで抽出
  3) SharePoint リスト「経費精算」に1行追加
       - 申請者 = 送信者メールアドレスで自動紐付け
       - ステータス = 下書き(読取に自信がない項目は「要確認」フラグ)
  4) 本人へ通知「一覧に追加しました。目的・同席者を補記してください」
      ▼
【マイページ(SharePoint ビュー / Power Apps)】
  本人は自分の行だけ表示・編集(アイテムレベル権限)
  「目的・費目・同席者(交際費のみ)」を補記 → 金額を目視確認 → 申請
      ▼
【承認フロー(Power Automate 承認)】
  上長承認 → 経理確認 → 承認済みは編集ロック
      ▼
【経理出力】
  月次で仕訳CSVを自動生成 → 会計ソフトにインポート
```

## 2. データ項目(SharePoint リスト「経費精算」)

| 列 | 入力元 | 備考 |
|---|---|---|
| 申請者 | 自動(送信者アドレス) | マイデータ紐付けキー |
| 利用日 | Claude自動抽出 | 本人が申請前に目視確認 |
| 支払先 | Claude自動抽出 | |
| 金額(税込) | Claude自動抽出 | **申請前に本人確認必須** |
| 税率内訳(10%/8%) | Claude自動抽出 | |
| インボイス登録番号(T+13桁) | Claude自動抽出 | 番号なし=免税事業者フラグ |
| 費目 | 本人補記(選択式) | 交通費/会議費/交際費/消耗品 等 |
| 目的・詳細 | 本人補記 | |
| 同席者 | 本人補記 | 交際費・会議費のみ必須 |
| 原本画像リンク | 自動 | ドキュメントライブラリへのリンク |
| ステータス | 自動+承認フロー | 下書き→申請→上長承認→経理確認→確定 |
| 読取確信度 | Claude出力 | 低確信は「要確認」表示(断定しない・憲法ルール準拠) |

## 4. 法務・税務要件(守り部レビュー必須)

紙の原本を破棄してスキャン画像を正本扱いにするには、**電子帳簿保存法のスキャナ保存要件**を満たす必要がある。

1. **改ざん防止**: タイムスタンプ付与、または「訂正削除の防止に関する事務処理規程」を整備(国税庁ひな形あり)。SharePointのバージョン履歴で変更履歴を保全
2. **検索要件**: 日付・金額・取引先で検索できること(上記リスト列で充足)
3. **入力期限**: 受領からおおむね7営業日以内(規程により最長2か月+7営業日)に撮影・取込
4. **画質**: 200dpi相当・カラー(スマホ撮影で通常充足。ブレ・見切れは差戻し)
5. インボイス制度: 登録番号の有無で仕入税額控除の扱いが変わるため、免税事業者フラグを経理に引き継ぐ

→ 最終的な要件充足の判断は**顧問税理士の確認を経てから運用開始**。それまでは紙原本を併存保管。

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| AIの読取誤り(特に金額) | 申請ボタン前に本人の目視確認を必須化。低確信度は「要確認」表示 |
| メール誤送信・添付漏れ | 受信後の自動返信で取込結果を本人に通知。失敗時は差戻し通知 |
| 電帳法要件の未充足 | 税理士確認まで紙原本併存。規程整備を運用開始の前提条件にする |
| 属人化 | フロー定義とこの設計書をリポジトリで管理。引き継ぎ台帳に登録 |
