## 3. セグメントタグ設計

会社情報登録(【会社情報登録】受信)を起点に、手動でタグ付与:

| タグ | 付与条件 |
|---|---|
| 地域:那覇 / 地域:それ以外 | 所在地から |
| 業種:(8分類) | 会社情報 or 会話から |
| 関心:承継 | 【事業承継相談】送信 or 承継系制度への反応 |
| 関心:専門家 | 【専門家相談】送信 |
| 状態:未登録情報 | 友だち追加のみで会社情報未登録 → 登録リマインド対象 |

## 4. 配信ルール(頻度と内容)

| 配信 | 頻度 | 内容 | ソース |
|---|---|---|---|
| 締切アラート | 週2まで(月/木 朝) | 締切7日以内の制度まとめ | **posts/line/alerts_latest.md(毎日自動生成)** |
| 新着まとめ | 週1(水) | 新規掲載の注目制度2〜3件 | data/subsidies.json の新着 |
| コラム | 月2まで | 申請準備(gBizID等)・活用事例 | SNS部と共用 |

原則: 週3通まで。ブロック率5%超の週は翌週の配信を1本減らす。
表現ルール: 誇大表現禁止・金額/締切は原文通り・必ず出典URL(絶対ルール#1)。

## 5. 締切アラートの運用手順(毎朝5分)

1. リポジトリの `posts/line/alerts_latest.md` を開く(毎日自動更新)
2. 「本文(コピペ用)」をコピー
3. LINE Official Account Manager → メッセージ配信 → 貼り付け → 配信(月/木のみ)
4. 「配信不要」と書かれていたらスキップ

自動配信に移行(report-hqで実証済みのGraph API+Power Automateの型を流用可能)。
移行判断は小柳さん決裁。

## 6. 応答メッセージ(自動応答に設定)

- 「【会社情報登録】…」受信時:
  「ありがとうございます!登録を承りました。貴社に合う制度が出ましたら優先してお知らせします🌺(通常1営業日以内に担当より確認のご連絡をする場合があります)」
- 「【専門家相談】」受信時:
  「承知しました!担当より1営業日以内にご連絡します。お急ぎの場合はこのままご相談内容をお送りください。」
- 「【事業承継相談】」受信時:
  「承知しました!承継専門の担当より1営業日以内にご連絡します。秘密は厳守します。」
