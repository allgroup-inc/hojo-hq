# フクギイロ 個人情報保護方針(ドラフト)

状態: **弁護士確認前・公開不可** / 起案: マモリ / 承認フロー: 三名体制議論 → うちむき担当 → 小柳さん決裁 → 弁護士確認 → 公開
関連: 診断部(ミチビキ)の世帯診断・伴走部(トドケ)のLINE配信の前提となる文書

## 設計原則(方針の方針)
1. **取得最小化**: 診断に必要な最小限だけを取得する。氏名・生年月日・住所番地・電話番号は**取得しない**
2. **本人にとって損のない設計**: 提供した情報が本人の不利益(勧誘・共有)に使われない構造を先に作る
3. **匿名で始められる**: 診断は匿名で完了できる。LINE登録は診断後の任意
4. **グループ共有は明示同意のみ**: ALLGROUP内の他サービス(保険・FP相談)への情報連携は、都度の明示同意がある場合に限る(暗黙のオプトインをしない)

## 取得する情報(診断)
| 項目 | 粒度 | 目的 |
|---|---|---|
| 居住市町村 | 市町村名まで(番地不要) | 市町村制度の判定 |
| 世帯構成 | 大人/子どもの人数・子の年齢帯 | 子育て・教育制度の判定 |
| ライフイベント | 選択式(妊娠・出産/失業/介護 等) | 該当制度の絞り込み |
| 収入帯 | 5区分程度の帯(任意回答) | 所得要件がある制度の目安判定 |
| LINE識別子 | LINE登録者のみ | 締切リマインド・受給フォロー配信 |

## 取得しない情報(明記する)
氏名・正確な住所・電話番号・マイナンバー・口座情報・病名等の要配慮個人情報(障がい・医療系制度は「該当の可能性がある窓口の案内」までとし、症状等の入力を求めない)

## 利用目的
1. 該当しうる制度の診断結果の表示
2. 締切リマインド・申請手順・受給確認のLINE配信(登録者のみ)
3. 匿名統計(市町村別・制度別の「届いていない支援」分析)の作成・公表・行政提供 — **個人を特定できない形に限る**
4. 明示同意がある場合のみ: 提携士業への紹介、ALLGROUP内サービスの案内

## 第三者提供・委託
- 原則しない。例外は①本人の明示同意(士業紹介等)②法令に基づく場合のみ
- システム委託先(LINE・クラウド等)には安全管理措置を確認する

## 保存期間・削除
- 診断データ: 取得から2年で自動削除(統計化済みの匿名データは除く)
- LINEブロック/退会時: 識別子に紐づくデータを90日以内に削除
- 本人からの削除請求: LINEまたは問い合わせ窓口で受け付け、30日以内に対応

## 安全管理
- アクセスは夫婦2名+必要最小限のシステムに限定。閲覧ログを保存しタダスさん(監査)が点検
- 個人データを含むファイルはリポジトリに置かない(リポジトリは制度DB・統計のみ)

## 三名体制議論の記録(起案時)
- スイシン: 匿名診断を核にすれば取得リスク自体が小さい。この設計で早期に弁護士確認へ
- ウタガイ: 収入帯は任意でも「答えないと診断精度が下がる」圧力になる。UI上「答えなくても診断できます」を明記すべき → 採用
- ベッカイ: そもそも収入帯を聞かず「所得要件がある制度は『要件あり』ラベルで全部見せる」設計もある。診断精度と取得最小化のトレードオフとしてA/Bで検証 → 採用(ミチビキの設計課題に登録)
- 裁定(マモリ): 上記2点を反映してドラフト確定。弁護士確認の論点は「要配慮情報の非取得設計の十分性」「グループ共有の同意取得方式」「LINE Platformの規約整合」の3点
