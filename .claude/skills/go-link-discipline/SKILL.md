# 中間リンク運用(go)

go(各SNS → `/go/<チャネル>` → GA4に経路記録 → LINE(lin.ee)へ自動転送)を
編集・追加するときは、README の仕組みを壊さない。

## 守るルール

1. **lin.ee を直接貼らない**: 必ず `/go/<チャネル>/` の中間リンクを経由する。
   直貼りすると ①経路計測 ②転送先の一括変更 ができなくなる。SNSプロフィール等には中間リンクを設定する。
2. **GA4計測を壊さない**: 各ページは転送前に `line_redirect` イベント + `channel` パラメータを記録する(約0.4秒)。
   この計測タグを削らない。測定ID(`G-XXXXXXXXXX`)は各ファイル2箇所を正しく置換する。
3. チャネルは ig / tt / yt / fb / booth / card / mv。追加するときも同じ「計測→転送」構造を踏襲する。

## 転送先(LINE)を変えるとき

- 各 `/go/<チャネル>/` の転送先を更新する(SNS側リンクは触らず、中間リンクの転送先だけ変える)。
- 変更後、主要チャネルで実際に転送されるか & GA4にイベントが飛ぶかを確認する。

## チャネル追加の手順

1. `/<チャネル>/` フォルダを作り、既存(例: ig)の計測+転送の雛形を踏襲する。
2. 測定IDを正しく設定し、`channel` パラメータをそのチャネル名にする。
3. GA4探索レポートに `line_redirect` × `channel` が出るか確認する。
