# コンプライアンス表現 & 承認ゲート(kakei-hq)

kakei-hq で利用者向けの文面(広告・LP・LINE・SNS・催事・チラシ等)を
作る/レビューするときは、憲法(CLAUDE.md)のコンプラ・承認ルールに従う。

## 表現ルール(全チャネル・全出力に適用)

- **断定表現は全チャネル禁止。** 「必ず損」「必ず得」「絶対に安くなる」等は使わない。
- 使ってよいのは **「〜なケースがあります」形式**(条件付き・可能性の提示)。
- 不安を過度に煽らない。敵は同業ではなく「無関心」という立ち位置を守る。

## チェック手順

1. 文面に断定表現(必ず/絶対/確実に得・損)が無いか確認する。あれば「〜なケースがあります」形へ直す。
2. 資格業務・金融助言に踏み込んでいないか確認する(有資格者の範囲を越えない)。
3. 数値・手数料・条件を載せる場合は根拠と条件を明記する。
4. 公開・支出を伴うものは **代表承認が必要**。承認ゲートは GitHub PR マージの1箇所のみ。窓口はえんくん。
   - 承認前に公開・配信・支出を実行しない。

## えんくんの行動原則(参考)

結論ファースト / 悪い数字ほど早く報告 / 提案には必ず代替案と根拠 / 代表の承認なしに公開・支出を実行しない。
