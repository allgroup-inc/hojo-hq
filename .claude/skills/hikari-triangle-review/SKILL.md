# 3名壁打ち(hikari-hq)

hikari-hq では施策実行前に、必ず部門内3名の視点を通す(組織憲法の壁打ち義務)。

## 3視点(部門ごとに役割は異なるが構造は共通)

- **攻め**: 前に進める。狙い・期待効果・想定CPA・実行案。
- **検証**: 数字と前提を検証する。根拠の穴・CVR/CPA試算・A/B観点。
- **守り**: リスク・規約・エリア・コンプラを見る。止めるべき理由を出す。

部門対応(CLAUDE.md): 広告運用=unyou/bunseki/risk、クリエイティブ=lp/shinri/ab、
インサイドセールス=sokko/tsuikyaku/talk、自動化=kiban/auto/kenshin、渉外=dairiten/ukezara/saisan。

## まとめ方(この形で出力)

```
## 施策: <テーマ>  / 想定CPA: <円>(根拠: ...)
### 攻め
- ...
### 検証   ← 空欄不可
- ...(反対・懸念・数字の穴)
### 守り
- ...(リスク・エリア・規約)
### 結論(エルくん or 部門長)
- 採否 / 理由 / 少数意見(2対1で割れた場合は必ず記録)
```

## 差し戻し条件

- **誰も反対しない案(全会一致)は前提を疑い直す。** 反対視点の記録が無い議事は無効。
- 2対1で割れたら、少数意見を記録した上でエルくんが判断する。
- CPA1,000円以下・エリア厳守(`hikari-cpa-guard`)を満たさない案は結論に進めない。
